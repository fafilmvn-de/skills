1 · What it does

You run a command. It uses your browser's SSO cookie to talk to Confluence, runs every page through a safety layer, and writes the result somewhere you can use. Hover any box for detail; click to jump to the source.

2 · Why it's safe to keep on disk

The defining feature of this tool. Any sufficiently large Confluence space contains pages that, written naively to disk, will be quarantined by Microsoft Defender for Endpoint or a similar on-access EDR. The companion failure is a recursive grep -r over the cache, which fans the AV scan out to every file and puts a keyword chain into command-line telemetry. The fix has two halves: every page write goes through this pipeline, and reads go through search instead of grep -r.

3 · What you can run

All commands are atlassian-sync <verb> after npm install -g . (or node scripts/sync-atlassian.mjs <verb> without install). The two highlighted rows are the ones to remember: search is how you read the cache, and never grep -r is the one rule.
VerbWhat it doesWhen you'd use it
setup Interactive base-URL + cookie acquisition. Writes ~/.atlassian-sync/.env. Once per machine. Then again with --renew when the cookie expires.
page <url> Single page → docs/<slug>.html (or .md with --md; checklist with --template checklist). "I want to share this one page" or "agent, read this page for context".
subtree <id> Recursive crawl from a root page → .confluence-cache/. Mirror a specific section of a space without crawling everything.
space <key> Full space crawl → .confluence-cache/. Full knowledge-base mirror. Slow first time, fast on repeat.
(no verb) Bulk sync from .atlassian-sync.json. Builds index at the end. Daily/weekly refresh of all configured spaces + root pages + Jira projects.
search "<q>" The only sanctioned way to read the cache. One file open, no fan-out. Accepts - to read from stdin. Every time you want to find something in the cache. Use this instead of grep.
index Force a rebuild of .confluence-cache/.index.json. After hand-editing the cache, or if search results look stale.
migrate Re-apply current safety rules to an existing cache. Rename-before-read so on-access AV stays quiet. After updating safety-rules.json, or to retrofit an old un-safe cache.
ping Quick auth + connectivity check. "Is my cookie still alive?"
grep -r …/.confluence-cache/ Don't. Triggers on-access AV across every page and lands a keyword chain in process telemetry that SIEM keyword rules flag as discovery. Never. Use search.
